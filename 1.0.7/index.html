    Clarity CLI üöÄ

    Interactive command-line tool for test management, profile handling, and component uploads to CyClarity.

    Why another CLI?
    Clarity CLI bundles everyday actions (login, test execution, artifact upload) into a single, script-friendly interface‚Äîso you can automate your QA pipelines without juggling multiple tools.

    üì¶ Installation

    pip install clarity-cli

    ‚ö° Quick start

    # 1Ô∏è‚É£  Configure a profile (once)
clarity profile-setup   --profile dev   --client-id <ID>   --client-secret <SECRET>   --token-endpoint https://auth.example.com/oauth/token   --scope "clarity offline_access"   --project 42 --workspace 7 --agent_id AGENT_1   --default

# 2Ô∏è‚É£  Log in (refreshes & stores JWT)
clarity login --profile dev

# 3Ô∏è‚É£  Run a single test
clarity execute --test-id TEST-123

    üóÇ Config files & profiles

    File / Dir Purpose Default location
    config.json Stores profiles & tokens ~/.clarity/config.json
    ~/.clarity/ Root config directory (auto-created) ‚Äì

    Profiles let you switch between environments (dev, staging, prod) without re-entering secrets.

    üõ† Command reference

    clarity execute

    Run a test or plan interactively or headless.

    Option Description Example
    --test-id, -t Execute a specific test. --test-id TEST-123
    --plan-id, -t Execute a specific plan. --plan-id PLAN-42
    --profile, -p Profile to use. --profile prod
    --override-config-path, -c Path to config.json. --override-config-path ./config.json
    --project, -proj Override project ID. --project 99
    --workspace, -w Override workspace ID. --workspace 5
    --agent-id, -a Run on specific agent. --agent-id AGENT_2
    --parameters-file Flow variable values (JSON). --parameters-file vars.json
    -y Non-interactive, assume yes. -y 
    clarity execute --test-id TEST-123 --parameters-file vars.json

    clarity profile-setup

    Create or update a profile.

    Option Required? Description
    --profile, -p ‚úî Profile name.
    --client-id, -id ‚úî OAuth client ID.
    --client-secret, -cs ‚úî OAuth client secret.
    --token-endpoint, -e ‚úî OAuth token URL.
    --scope, -s ‚úî OAuth scopes (space-separated).
    --project, -proj ‚Äì Default project ID.
    --workspace, -w ‚Äì Default workspace ID.
    --agent_id, -a ‚Äì Default agent ID.
    --domain ‚Äì Override Clarity domain.
    --default, -d ‚Äì Set as default profile. 
    clarity profile-setup -p staging -id XXX -cs YYY -e https://auth/staging/token -s "clarity"

    clarity login

    Log in with a saved profile and retrieve a new token.

    Option Description
    --profile, -p Profile to use (fallback to default).
    --override-config-path, -c Path to config.json. 
    clarity login -p prod

    clarity upload

    Package & upload a component to CyClarity.

    Option Description
    --profile, -p Profile to use.
    --override-config-path, -c Path to config.json.
    --component-path Path to Poetry project.
    --entrypoint, -e Format: module:function.
    --running-env, -r Target runtime: IOT or CLOUD.
    --pyc Compile .pyc before upload.
    -y Non-interactive, assume yes. 
    clarity upload --component-path ./my_component --entrypoint my_component.main:handler -r CLOUD --pyc -y

    clarity get-findings

    Retrieve and display findings from a specific test execution with pagination support.

    Option Description
    --execution-id, -e Execution ID to get findings for (required if not interactive).
    --output-type, -o Output format: table (default) or json.
    --file-path, -f Optional file path to save findings data as JSON.
    --page-size, -s Number of findings per page (default: 20, use 0 for no pagination).
    --profile, -p Profile to use (fallback to default).
    --override-config-path, -c Path to config.json.

    Features

    • Smart Pagination: Shows first 20 findings in terminal, saves remaining to file
    • Interactive Navigation: When using pagination, navigate with simple commands
    • Multiple Output Formats: Table view for terminal, JSON for automation
    • File Export: Automatically saves large result sets to timestamped JSON files

    Examples

    # Basic usage - shows first 20 findings in table format
clarity get-findings --execution-id TestExecution_123

# Get all findings without pagination
clarity get-findings --execution-id TestExecution_123 --page-size 0

# Export to JSON file
clarity get-findings --execution-id TestExecution_123 --output-type json --file-path findings.json

    Pagination Behavior

    When you have more than 20 findings (or your specified page size):

    1. Terminal Display: Shows first N findings in a formatted table
    2. Automatic File Save: Remaining findings saved to findings_ExecutionID_timestamp.json
    3. Clickable Link: Provides file path for easy access to complete data

    Report Generation Guide

    This guide explains how to use the generate-report command and understand the different response scenarios for automation purposes.

    Command Usage

    clarity generate-report [OPTIONS]

    Options

    • --report-type, -r [PDF|CSV]: Report format (default: PDF)
    • --execution-id, -e TEXT: Specify the execution ID to generate report for
    • --project-id, -proj TEXT: Specify the project ID
    • --profile, -p TEXT: Profile to use
    • --override-config-path, -cfg TEXT: Path to config.json

    Examples

    # Generate PDF report (default)
clarity generate-report --execution-id TestExecution_123 --project-id TestProj --report-type PDF --profile ""

# Generate CSV report
clarity generate-report --execution-id TestExecution_123 --project-id TestProj --report-type CSV --profile ""

# Interactive mode (will prompt for missing parameters)
clarity generate-report --report-type PDF

    Response Scenarios

    The system returns different responses based on the current status of the report generation process.

    1. Report Completed ‚úÖ

    When: Report generation is finished and file is available for download

    Response Format:

    {
  "statusCode": 200,
  "body": {
    "presigned_url_link": "https://s3-presigned-url-to-download-file"
  }
}

    CLI Output:

    ‚úÖ Report generation completed!
üìÑ PDF Report is ready!
Download URL: https://s3-presigned-url-to-download-file
You can copy and paste this URL into your browser

    For Automation: - Condition: Response contains presigned_url_link with actual URL - Action: Download the file using the provided URL - The URL is a time-limited presigned S3 URL

    2. Report In Progress üîÑ

    When: Report generation has been triggered and is currently running

    Response Format:

    {
  "statusCode": 200,
  "body": {
    "message": "Report generation in process."
  }
}

    CLI Output:

    ‚ö†Ô∏è  Report generation in process.
You can run this command again later to check the status

    For Automation: - Condition: Response contains message field (typically "Report generation in process.") - Action: Wait and retry the command after some time (recommended: 30-60 seconds) - Continue polling until you get a presigned URL or failure message

    3. Report Not Started üìã

    When: No report generation has been initiated for this execution

    Response Format:

    {
  "statusCode": 200,
  "body": {
    "presigned_url_link": ""
  }
}

    CLI Output:

    ‚ö†Ô∏è  Report is not ready yet. No download link available.
You can run this command again later to check the status

    For Automation: - Condition: Response contains presigned_url_link with empty string value - Action: The CLI will automatically trigger report generation - After triggering, subsequent checks will show "Report generation in process."

    4. Report Failed ‚ùå

    When: Report generation encountered an error and failed

    Response Format:

    {
  "statusCode": 400,
  "body": {
    "message": "Report generation has failed."
  }
}

    CLI Output:

    ‚ùå Report generation has failed.

    For Automation: - Condition: Response has statusCode: 400 with error message - Action: Check execution logs or retry the command - May require manual intervention

    Automation Workflow

    Here's a recommended automation workflow:

    Python Example

    import subprocess
import json
import time

def wait_for_report(execution_id, project_id,  profile, report_type):
    """
    Wait for report generation to complete and return download URL
    Continues indefinitely until report is ready or fails
    """
    attempt = 0

    while True:
        attempt += 1
        try:
            # Run the CLI command
            result = subprocess.run([
                'clarity', 'generate-report',
                '--profile', profile,
                '--execution-id', execution_id,
                '--project-id', project_id,
                '--report-type', report_type
            ], capture_output=True, text=True)

            # Check the actual CLI output messages (regardless of return code)
            output = result.stdout + result.stderr

            if "Report is ready" in output or "Download URL:" in output:
                # Report completed successfully
                print("Report ready! Check CLI output for download URL")
                print(f"Output: {output}")
                return True
            elif "Report generation in process" in output:
                print(f"Attempt {attempt}: Report still generating...")
                time.sleep(30)  # Wait 30 seconds
            elif "Report generation has failed" in output:
                print("Report generation failed!")
                return False
            elif "Report is not ready yet" in output:
                print(
                    f"Attempt {attempt}: Report not started, will be triggered...")
                time.sleep(15)  # Wait 15 seconds
            elif "Generating" in output and "Report" in output:
                # This handles the initial "Generating PDF Report" message
                print(f"Attempt {attempt}: Starting report generation...")
                time.sleep(15)  # Wait 15 seconds
            else:
                print(f"Attempt {attempt}: Checking status...")
                print(f"Output: {output}")
                time.sleep(15)  # Wait and try again

        except KeyboardInterrupt:
            print("\nStopped by user (Ctrl+C)")
            return False
        except Exception as e:
            print(f"Error running command: {e}")
            return False

# Usage
if wait_for_report("TestExecution_123", "TestProj", "PDF"):
    print("Success! Download the report using the provided URL")
else:
    print("Failed to generate report or stopped by user")

    Report Types

    PDF Reports

    • Format: Portable Document Format
    • Content: Comprehensive test execution report with findings, charts, and detailed analysis
    • Use Case: Human-readable reports, presentations, documentation
    • File Extension: .pdf

    CSV Reports

    • Format: Comma-Separated Values
    • Content: Structured data export of findings and execution details
    • Use Case: Data analysis, integration with other tools, spreadsheet import
    • File Extension: .csv
